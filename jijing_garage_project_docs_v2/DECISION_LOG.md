# DECISION_LOG.md

## 目的

本文件用于记录项目关键技术和产品决策，避免 Agent 或开发者反复改变方向。

## 决策记录

### Decision 001：第一版使用微信小程序原生开发

决定：

```text
使用 WXML + WXSS + JavaScript + JSON
```

原因：

```text
项目第一阶段功能较轻，原生小程序开发最直接，方便快速预览和上线。
```

不采用：

```text
Taro
UniApp
Vue
React
TypeScript
第三方 UI 框架
```

---

### Decision 002：第一版使用本地 mock data

决定：

```text
车辆和分类数据先放在 data/cars.js 和 data/categories.js
```

原因：

```text
先完成展示和交互闭环，避免过早接入云开发导致开发复杂度上升。
```

---

### Decision 003：第一版不做交易系统

决定：

```text
只做预约咨询，不做支付、押金、合同、真实订单。
```

原因：

```text
租车业务涉及身份、驾驶证、押金、合同、保险等复杂合规流程，第一版先人工确认更稳。
```

---

### Decision 004：首页风格锁定为深色车库风

决定：

```text
黑色背景、大图车辆卡片、状态标签、价格展示。
```

原因：

```text
需要接近用户提供的参考截图，突出车库感和高质感展示。
```

---

### Decision 005：所有车辆跳转使用 carId

决定：

```text
详情页和预约页都通过 carId 查询车辆。
```

原因：

```text
车辆名称可能重复，数组下标不稳定，carId 最适合作为唯一标识。
```
